### Iteration 2/5 - 模型路由調整

### Iteration 3/5 - Response 緩存

**🧪 假設：**
```
實施 response 緩存系統，對於重複查詢直接返回緩存結果，
可以節省 15-25% Token 用量。
```

**🔬 緩存策略：**

```javascript
{
  "cache": {
    "enabled": true,
    "ttl": 3600, // 1 小時
    "maxSize": 1000, // 最多 1000 條
    "excludePatterns": ["當前時間", "天氣", "新聞"] // 動態內容不緩存
  }
}
```

**緩存場景示例：**
```
用戶問：「OpenClaw 點樣安裝？」
→ 第一次：正常調用 LLM (消耗 Token)
→ 第二次 (1 小時內)：返回緩存 (0 Token)
```

**預計節省：**
- 重複查詢 (約 20%): 100% Token 節省
- 總節省：20% × 100% = 20%

### Iteration 4/5 - 減少冗餘上下文

**🧪 假設：**
```
優化對話上下文管理，只保留必要信息，
可以節省 5-8% Token 用量。
```

**🔬 上下文策略：**

```javascript
{

    "excludeSystemMessages": true // 排除系統消息
  }
}
```
