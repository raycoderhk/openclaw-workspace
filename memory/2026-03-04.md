## 📊 Technical Decisions

### User ID Mapping
**Decision:** Use email as stable identifier (same as Kanban Board)

**Reason:** Google ID is string, Supabase expects UUID. Email is consistent across both systems.

**Implementation:**
```typescript
async function getOrCreateUser(email, name, googleId, image) {
    // Find by email first
    let user = await supabase.from('users').select('id').eq('email', email).single();
    
    // Create if doesn't exist
    if (!user) {
        user = await supabase.from('users').insert({ email, name, google_id: googleId }).select('id').single();
    }
    
    return user.id; // UUID for queries
}
```

### Database Design
**Decision:** Separate tables for each widget type

**Reason:** 
- Clear data separation
- Easier to add RLS policies per table
- Better query performance
- Simpler to extend with new widgets

**Tables:**
- `user_events` → Calendar widget
- `user_goals` → Goal tracker widget
- `user_friends` → Friends network widget
- `user_settings` → Dashboard customization
